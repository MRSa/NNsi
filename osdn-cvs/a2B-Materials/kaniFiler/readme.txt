■kaniFilerとは？
MIDP 2.0 (JSR75)対応のファイラーです。簡単なテキスト編集も可能です。
ファイルに書かれているURLからファイルを取得することもできます。
Willcomの端末、WX310SA/WX310J, WX320Kなどで動作します。

■1.7.7での変更点
 - 「テキスト編集」機能で、ブラウザで開く機能に、Google Mobile Proxy
  経由でリンクを開くことができるようにする。

■機能
以下の機能があります。
 - ディレクトリの作成
 - ファイル/フォルダのリネーム
 - ファイルの複製
 - (複数)ファイルのコピー
 - (複数)ファイルの削除
 - 空ディレクトリの削除
 - ファイルをWebブラウザで開く (1.1.0以降)
 - ファイルの属性変更 (書き込み禁止属性、隠しファイル属性、1.2.0以降)
 - テキストファイルの編集 (1.2.0以降)
 - ディレクトリのファイル一覧(htmlファイル)出力 (1.3.0以降)
 - テキストファイル編集中、ファイル内のURLを使用して「ブラウザで開く」(1.4.0以降)
 - テキストファイル編集中、ファイル内のURLを使用して「URLを取得」(1.4.0以降)
 - 画像の表示(1.5.0以降)
 - サウンドの再生(1.6.0以降)
 - サウンドの停止(1.6.1以降)
 - ZIPファイルの展開(1.7.0以降)

■操作
以下、WX310SAのキーで説明を行います。
 - カーソルキー上下左右でアイテムを移動します。
 - 文字キーでひとつ上の階層に移動します。
 - フォルダにカーソルがあったとき、Myキーを押すと一つ下の階層に移動します。
 - MENUキーでファイルの選択/解除を行います。
 - フォルダにカーソルがあったときにMENUキーを押すと階層移動を行います。
 - *キー、または#キーを押すとファイルの日付、ファイルサイズを画面下部に表示します。 
 - MENUキー長押しでフォルダ内にある全ファイルの選択ができます。
 - CLRキーでファイルの選択解除ができます。
 - CLRキー長押しでフォルダ内にある全ファイルの選択解除ができます。
 - 実行(メール)ボタンで、操作の選択モードになります。(CLRキーで戻ります。)
  → "Dir作成" は、フォルダ(ディレクトリ)の作成ができます。
  → "リネーム"は、選択したファイル名の変更ができます。
  → "複製"は、選択したファイルを別名で複製できます。
  → "コピー"は、選択したファイルを別のディレクトリにコピーできます。
  → "削除"は、選択したファイル/フォルダを削除できます。
  → "ブラウザで開く"は、kaniFilerを終了し、選択したファイルをネットフロントで開きます。
  → "テキスト編集"は、選択したファイルをテキスト編集します。

■テキスト編集モードについて
ファイルのテキスト編集モードですが、Javaの制約により、１行づつの編集になります。
画面上部に、[XXXX:YYYY] の表示がありますが、XXXXが現在カーソルがいる行、
YYYYがファイル全体の行数を示します。

□キー操作
 ・ カーソルキー上下で前行/次行、左右で左右に移動します。
 ・ ▲キーで前ページ、▼キーで次ページに移動します。
 ・ MENUキーで行編集モードになります。
  → 編集後、『確定』を押すとその行を変更します。
  → 『Cancel』を押すと、その行を変更せずに戻ります。
 ・ * キーで、ファイルの先頭に移動します。
 ・ # キーで、ファイルの末尾に移動します。
 ・CLRキーで、テキスト編集モードから抜けます。
   → ファイルを編集していた場合、ファイルを保存して終了するかどうか確認をします。
   → 「実行」を選択するとファイルを保存した後、ファイル選択モードに移ります。
   → 「中止」を選択するとファイルを保存せずにファイル選択モードに移ります。
   → CLRキーを押すと、ファイルを保存せずにファイル選択モードに移ります。
 ・実行(メール)ボタンで、編集操作の選択モードになります。
   → 「保存」は、ファイルの保存を行います。
   → 「次行挿入」は、カーソルがいる行の次の行に空行を挿入します。
   → 「1行挿入」は、カーソルがいる行に空行を挿入します。
   → 「1行貼り付け」は、前回"1行削除"した行をカーソルがいる行に挿入します。
   → 「1行削除」は、カーソルがいる行を削除します。

□ファイルの保存について
以前のファイルを(ファイル名).bak とリネームし、同じファイル名で保存します。
（重要なファイルを編集する場合、あらかじめ複製を作成してから編集してください。）

■現バージョンでの制限
 - ファイル操作エラーの表示は行いません。
 - 空でないディレクトリは削除できません。
 - テキスト編集は新規作成することができません。
 - テキスト編集時、文字コードおよび改行コードについて考慮していません。
 - ZIPファイルの展開は、かなり遅い場合があります。（WX310SAでは遅いことを確認しています。）

■変更履歴
□1.7.6での変更点
 - 「file一覧出力」機能で、対応機種の場合、末尾にkaniFilerを起動するリンクを
  出力するようにする。(file一覧ファイルをブラウザで開いた後、kaniFilerへ戻って
  くることができるようになります。)

□1.7.5での変更点
 - WX340K/WX341K(WS009KEも?)からの「ブラウザで開く」を対応する。

□1.7.4での変更点
 - WS023T(WS018KEも?)からの「ブラウザで開く」を対応する。

□1.7.3での変更点
 - ZIPファイルの展開機能で、ファイル展開に失敗する場合があったのを再度修正する。

□1.7.2での変更点
 - ZIPファイルの展開機能で、ファイル展開に失敗する場合があったのを修正する。

□1.7.1での変更点
 - ZIPファイルの展開機能で、ディレクトリを作成せず、ZIPファイルのある
  ディレクトリにファイルを展開できるようにする。
  → コマンドメニュー "ZIP展開(同Dir)を追加しました。
 - kaniFilerをコマンドメニューから終了できるようにする。
 - コマンドメニューを２行表示に変更する。

□1.7.0での変更点
 - サウンドの再生・停止機能をトグルに変更する。
  → サウンドファイルを選択中は、サウンドを再生します。
  → サウンドファイル以外を選択中の場合にはサウンドを停止します。
 - ZIPファイルの展開機能を追加する。
  → 機種によっては、メチャクチャ遅いかもしれません。。。ごめんなさい。
  
  ※ ZIPファイルの展開機能は、JZlib 1.0.7 ( http://www.jcraft.com/jzlib/ )を
    利用して実装しました。
     → 2ちゃんねるの "willcom Javaアプリスレッド vol.5" スレ34番さんに存在を
       教わりました。ありがとうございました。

□1.6.1での変更点
 - 再生中のサウンドを停止できるようにする。

□1.6.0での変更点
 - .mld/.mid ファイルはMIDIファイルとしてサウンド再生できるようにする。
 - .wavファイルは、WAVEファイルとしてサウンド再生できるようにする。

□1.5.3での変更点
 - WX320Kではローカルファイルを「ブラウザで開く」機能が動かなかった
  不具合を修正する。
 - GAME Dキーの長押しに「ブラウザで開く」機能を割り当てる。

□1.5.2での変更点
 - 画像表示モードで、JPEGファイルの画像を表示しているとき、数字キーの
  ゼロ（０）を押すと、本体画像の表示を行うようにトライしてみるようにする。
  → メモリが不足していた場合、JPEGファイルのサムネールデータ表示の
     ままとなります。
  → 一度数字キーのゼロを押すと、本体画像の表示をトライするモードか、
    JPEGファイルのサムネールデータを表示するか、を切り替えます。
    (kaniFilerを終了させると、モードをクリアします。)

□1.5.1での変更点
 - 画像表示モードのとき、JPEGファイルでの表示をExifの縮小画像を
  使用して表示できるかトライしてみるよう変更する。
 → デジカメで撮影したJPEGファイルを表示できるようになった、かもしれません。

 - 画像表示モードで、画像が小さい場合には ウィンドウの真ん中に表示するように
  変更する。

□1.5.0での変更点
 - 画像を表示できるようにしてみる。
  → jpegファイル、あるいはpngファイルを選択し、GAME Dを押す、あるいは
    メニューの「画像表示」を押すと表示します。
　→ 画像表示中に上下左右キーで表示場所を移動させることができます。
  → しかし、メモリサイズ等、いろんな理由から表示できない可能性もあります。
    あしからずご了承ください。(480x320 程度が最大かも。。)
 - ファイル一覧表示モードで、GAME Aをカーソルを3つ上に移動させ、
  GAME Bをカーソルを３つ下に移動させるよう仕様を変更する。
 - ファイル一覧表示モードで、*キーでディレクトリの先頭、#キーでディレクトリの
  末尾に移動するようにする。 0キーで、ファイルの情報を表示するようにする。
  → 移動は、a2Bにあわせました。
 - 画像表示などを行った場合に、カーソルの場所を移動させないようにしてみる。
 - ディレクトリ一覧で、GAME Dを押すと画像表示およびテキスト編集ができるようにする。

□1.4.0での変更点
 - アイコンを作ってみる。
 - テキストファイル編集時に、「ブラウザで開く」と「URLを取得」機能を追加しました。
  → テキストファイル編集中の「実行」メニューにこのコマンドを追加しました。
  → 選択している行にあるURLに対して操作します。
  → 「URLを取得」の場合には、今編集中のテキストファイルと同じディレクトリに
     ファイルを作成します。
  → 通信終了後、バイブ(振動)によって取得が終了したことを通知します。
    （画面表示には何も反映させていません。ご了承ください。）
  → 取得するファイルは、確認せず上書きを実行します。ご注意ください。

□1.3.0での変更点
 - 「ファイル一覧出力」機能を追加する。
  → カレントディレクトリのファイル一覧をhtmlファイル(fileList.html)に出力します。
  → ファイル出力後、fileList.htmlを『ブラウザで開く』すると、ブラウザで参照できます。

□1.2.1での変更点
 - 終了時のディレクトリが削除されていたとき、再起動すると何も
  できなかった不具合を修正。

□1.2.0での変更点
 - ファイル/ディレクトリの属性変更機能を追加する
  → 書き込み禁止属性、隠しファイル属性の設定/設定解除が可能です。
 - テキストファイルの編集機能を追加する
  → 1行単位での編集が可能です。
