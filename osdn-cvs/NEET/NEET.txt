■はじめに
  NEETは、「NNsiが使用するデータベースファイルを母艦ＰＣで保守(メンテナンス)
できるようにする」ことを目的として作成しました。

  まだあまり機能がありませんが、徐々に拡充させていきたいと思います。
NNsiのコンジット、NNsiMonaともどもよろしくお願いいたします。


■前バージョン(NEET Phase2 Ver.0.20r000)からの変更点

  ・NNsi Ver.1.24の参照ログ取得での「取得ログ加工」モードの設定に対応。


■動作環境
  NEETを動作させるには、以下の環境が必須です。別途お使いの母艦ＰＣにインス
トールをお願いします。

  ・Microsoft .NET Framework
  ・par.exe
    → 詳細は、http://www.djw.org/product/palm/par/ を 参照してください。 
      ( http://www.djw.org/product/palm/par/parwin32.zip )


■添付ファイル
  ・NEET.txt
    → 本ファイルです。NEETの簡単な説明を記述しています。
  ・NEET.exe
    → NEETの本体ファイルです。.NET Framework上で動作するアプリケーションです。
  ・LogPermanent-NNsi_empty.pdb
    → NNsiの空の参照ログＤＢファイルです。１からサイトを登録する場合に
      ご利用いただけたら、と思います。
  ・LogToken-NNsi_empty.pdb
    → NNsiの空の区切り文字列ＤＢファイルです。１から区切りデータを登録する
      場合にご利用いただけたら、と思います。
  ・NGWord-NNsi_empty.pdb
    → NNsiの空のNGワード文字列ＤＢファイルです。１からNGワードを登録する
      場合にご利用いただけたら、と思います。

■使用方法

◇起動
  NEET.exe を起動してください。

◇設定
  NEET.exeを起動すると、par.exe の存在するディレクトリを指定するウィンドウが
 開きます。編集したいDBのボタンを押し、ファイル選択ウィンドウが表示されます
 ので、編集したいDBファイルを指定してください。
 (その後、一覧ウィンドウが開きます。)

  ※※[[ 注意 ]]※※
   1. ファイルの場所は、『スペースのないディレクトリ指定』が必要です。
     (現状の制約事項です。ご了承ください。)

   2. 区切り文字列は仮登録です。使用するためには、NNsiの『参照ログ・データ
     区切設定』で確認後、使用可能です。

◇一覧表示
  ファイル設定ダイアログで「決定」ボタンを押すと、参照ログＤＢを読み込み、
現在の登録済み参照ログサイト一覧を表示します。

◇サイトの登録と編集
  「レコード編集...」または「新規レコード...」を押すと、編集ダイアログを
表示します。（このダイアログは、NNsiに搭載している参照ログ取得設定ダイア
ログと同等です。）
  ※ 「試しに開く」ボタンで、ＵＲＬが妥当かどうかチェック可能です。

◇ＤＢファイルの出力
  一覧表示で「ファイル保存」を押すと、保存ダイアログを表示します。ＤＢ
ファイルを指定し、保存を行って下さい。

  ※※[[ 注意 ]]※※
   1. 出力するファイルの場所は、『スペースのないディレクトリ指定』が必要です。
     (現状の制約事項です。ご了承ください。)

   2. NNsiにＤＢファイルを転送するときには、DBファイル名にご注意ください。
     → 参照ログDB     : LogPermanent-NNsi.pdb
     → 区切り文字列   : LogToken-NNsi.pdb
     → NGワード文字列 : NGWord-NNsi.pdb
     → 板一覧DB       : BBS-NNsi.pdb

◇終了
  ファイル設定ダイアログを終了させることでNEETが終了します。
